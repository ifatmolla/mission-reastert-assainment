১. null এবং undefined এর মধ্যে পার্থক্য কী?
undefined: যখন একটি ভেরিয়েবল ডিক্লেয়ার করা হয় কিন্তু তাতে কোনো ভ্যালু দেওয়া হয় না, তখন জাভাস্ক্রিপ্ট ডিফল্টভাবে তাকে undefined হিসেবে সেট করে। এর মানে হলো ভেরিয়েবলটি আছে কিন্তু এর কোনো মান নেই।

null: এটি একটি অ্যাসাইনমেন্ট ভ্যালু। এটি প্রোগ্রামার দ্বারা ইচ্ছাকৃতভাবে ব্যবহার করা হয় বোঝানোর জন্য যে ভেরিয়েবলটি বর্তমানে "খালি" বা "শূন্য"।

২. map() ফাংশনের কাজ কী? এটি forEach() থেকে কীভাবে আলাদা?
map(): এটি একটি অ্যারের প্রতিটি উপাদানের ওপর কাজ করে একটি নতুন অ্যারে তৈরি করে এবং সেটি রিটার্ন করে। অরিজিনাল অ্যারেতে কোনো পরিবর্তন আনে না। (এটি চেইনিং করার জন্য উপযুক্ত)।

forEach(): এটি অ্যারের প্রতিটি উপাদানের ওপর লুপ চালায় এবং নির্দিষ্ট কাজ সম্পন্ন করে (যেমন: console.log করা)। এটি কোনো নতুন অ্যারে রিটার্ন করে না।

৩. == এবং === এর মধ্যে পার্থক্য কী?
== (Abstract Equality): এটি শুধুমাত্র ভ্যালু (মান) চেক করে। যদি ডেটা টাইপ আলাদা হয়, তবে এটি টাইপ কনভার্ট করে চেক করে। (যেমন: 5 == '5' সঠিক বা true)

=== (Strict Equality): এটি ভ্যালু এবং ডেটা টাইপ—উভয়ই চেক করে। কোনো অটোমেটিক টাইপ কনভার্সন করে না। (যেমন: 5 === '5' ভুল বা false)

৪. API ফেচ করার ক্ষেত্রে async/await এর গুরুত্ব কী?
জাভাস্ক্রিপ্ট সাধারণত সিনক্রোনাসলি কাজ করে। কিন্তু API থেকে ডেটা আসতে সময় লাগে।

Significance: async/await ব্যবহার করলে কোড দেখতে সিনক্রোনাস বা সাধারণ কোডের মতো লাগে, কিন্তু এটি ব্যাকগ্রাউন্ডে অ্যাসিনক্রোনাস কাজ সম্পন্ন করে। এটি আমাদের "Promise Hell" বা জটিল .then() চেইন থেকে মুক্তি দেয় এবং এরর হ্যান্ডলিং (try/catch) সহজ করে।

৫. জাভাস্ক্রিপ্টে Scope (স্কোপ) এর ধারণা
স্কোপ ঠিক করে দেয় একটি ভেরিয়েবল কোডের কোথায় ব্যবহার করা যাবে এবং কোথায় যাবে না।

Global Scope: ফাংশনের বাইরে ডিক্লেয়ার করা ভেরিয়েবল, যা পুরো স্ক্রিপ্টের যেকোনো জায়গা থেকে অ্যাক্সেস করা যায়।

Function Scope: একটি ফাংশনের ভেতরে ডিক্লেয়ার করা ভেরিয়েবল যা শুধুমাত্র ওই ফাংশনের ভেতরেই কাজ করে।

Block Scope: let এবং const দিয়ে ডিক্লেয়ার করা ভেরিয়েবল যা শুধুমাত্র একটি নির্দিষ্ট ব্লকের { ... } (যেমন: if বা for লুপ) ভেতরে সীমাবদ্ধ থাকে।
